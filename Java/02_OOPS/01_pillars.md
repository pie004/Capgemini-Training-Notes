**OOPS (Object-Oriented Programming System)** is a programming approach that organizes software design around **data and behavior** rather than functions and logic. It helps in building modular, reusable, and scalable applications by modeling real-world entities.

---

## 4 Pillars of OOPS:

### 1. Encapsulation

Encapsulation means **wrapping data and methods together into a single unit** and restricting direct access to data.
- Achieved using access modifiers (private, public, etc.)
- Improves data security and control

---

### 2. Abstraction

Abstraction means **hiding internal implementation details** and showing only essential features to the user.
- Reduces complexity
- Focuses only on what is necessary

---

### 3. Inheritance

Inheritance allows one entity to **acquire properties and behavior of another**, promoting code reuse.
- Supports hierarchical classification
- Reduces redundancy

---

### 4. Polymorphism

Polymorphism means **one interface, multiple forms**.
- Same operation behaves differently in different situations
- Achieved through method overloading and method overriding
